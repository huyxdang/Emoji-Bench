The Idea

Most verification benchmarks can be gamed by re-execution.

The standard way to test whether a model can check a reasoning chain: give it a derivation and ask whether it's correct. But a model can pass this without reading the steps at all — just re-execute the chain from scratch, compare the result to the claimed endpoint, and flag any mismatch. This looks like verification. It isn't.

Emoji-Bench isolates the case where this shortcut breaks. A reconvergent error is a derivation where one intermediate step violates the rules, but the chain recovers and reaches the correct final answer anyway. The endpoint looks fine. Only step-level attention catches the error.

Re-execution strategy
Step 1: valid
Step 2: rule violated ←
Step 3: locally valid
Final: 🐙 (correct endpoint)
Re-run → same answer → "no error" ✗
Step-level verification
Step 1: valid ✓
Step 2: 🚨 rule violated
Step 3
Final: 🐙
"Error at step 2." ✓

The formal systems use emoji as their symbols — not numbers, algebra, or any domain a model has seen before. Every rule must be read from the tables provided in the prompt. There is no pattern to recall, no shortcut to take.

What We Test

Four error types. One adversarial anti-shortcut.

The benchmark defines four kinds of errors. The first three are useful baselines. The fourth — E-RECONV — is the one designed to defeat re-execution.

E-RES
Result Error

A valid rule is cited correctly, but the output is computed wrong. The step references a real rule; the result is just off.

E-INV
Invalid Rule

The step cites a rule that does not exist in the formal system. The citation looks plausible but has no entry in the operation tables.

E-CASC
Cascade Error

An early step is wrong. Later steps are locally valid given that wrong intermediate value, but the final answer is incorrect. The error propagates.

E-RECONV
Reconvergent Error

An early step violates the rules, but the chain still reaches the correct final answer. The endpoint is indistinguishable from a valid derivation. Only step-level checking will catch it.

The Finding

Detection and localization diverge sharply.

Two metrics matter. Detection rate: does the model say an error exists? Step accuracy: does it identify the exact first wrong step? For models using re-execution shortcuts, these diverge. Models with extended thinking close the gap.
