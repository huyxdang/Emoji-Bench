Design Choices

What the benchmark isolates.

Novel formal systems, not familiar domains

Existing metacognition benchmarks sit inside arithmetic, algebra, or code — all heavily represented in pretraining. Emoji-Bench invents a fresh symbolic world for each instance. If the algebra never existed before the prompt, detecting an error requires actual table lookup rather than learned error aesthetics.

Emoji symbols are deliberate

The symbol pool is curated for single-token behavior across major model families, clear visual separation, and zero mathematical baggage. A flamingo or a magnet doesn't smuggle arithmetic priors into the task.

Difficulty scales by rule load, not chain length

Every chain stays short while the formal system gets denser. The task gets harder because there is more to track, not because the prompt becomes a wall of steps.

Error injection probes different failure modes

E-RES tests whether a model catches a wrong output. E-INV tests whether it checks that a cited rule exists. E-CASC makes later steps locally valid after an early root mistake. Together they distinguish surface fluency from real verification.

Dataset

Four deterministic stages.

Each benchmark instance is built from a seeded pipeline. The dataset is fully reproducible.

01

System generation

Sample 3–6 emoji symbols, generate operation tables, optionally add derived ops and distributing transformations.

02

Chain construction

Build a leftmost-innermost derivation so each reduction step is explicit and evaluation order is unambiguous.

03

Error injection

Emit a clean version plus up to three single-error variants. Cascading errors are recomputed downstream.

04

Validation

Check closure, table completeness, derived-op consistency, and transformation distribution.

Example

What a failure case looks like.

A medium-difficulty E-CASC instance. Step 2 is wrong, and the rest of the chain is recomputed from that wrong state. 

Symbols: {🪸, 🪩, 🧄, 🍄}

Derived op: x ⊗ y = (x ⊕ x) ⊕ y
Transform: inv(🪸)=🪩, inv(🪩)=🪸, inv(🧄)=🍄, inv(🍄)=🧄

Start: (inv(🧄) ⊗ inv(🍄))

Step 1: (inv(🧄) ⊗ inv(🍄)) → (🍄 ⊗ inv(🍄))   [by inv]
Step 2: (🍄 ⊗ inv(🍄)) → (🍄 ⊗ 🍄)             [by inv]  ← first error
Step 3: (🍄 ⊗ 🍄) → ((🍄 ⊕ 🍄) ⊕ 🍄)           [by definition of ⊗]
Step 4: ((🍄 ⊕ 🍄) ⊕ 🍄) → (🪩 ⊕ 🍄)           [by ⊕ table]
Step 5: (🪩 ⊕ 🍄) → 🪸                           [by ⊕ table]

A shallow checker sees steps 3–5 as a coherent derivation. But step 2 applied the transform incorrectly. E-CASC punishes local plausibility heuristics and rewards verification from the start of the chain.

Protocol

Prompting, outputs, and scoring.

Prompt structure

Each prompt contains the rules, the worked derivation, and a task asking whether there is an error and what the first incorrect step is. The wording "your working out" frames the task as metacognitive auditing.

Output format

All evaluations use structured output. OpenAI uses the Responses API parse path, Anthropic uses Messages API JSON schema, and Mistral uses JSON object response formatting at temperature zero.

Primary metric

Joint accuracy: the model must both decide correctly whether an error exists and identify the first wrong step. The false positive slice is included because some models look competitive only by hallucinating errors on clean rows.

Takeaways

What the first run suggests.

  • GPT-5.4 is the clear outlier

    It leads every aggregate slice and posts a 0.0% false positive rate on clean chains. That combination suggests systematic verification rather than broad suspicion.

  • Invented-rule errors are the real separator

    E-INV exposes whether a model checks that a rule exists instead of merely validating the resulting rewrite. Several otherwise strong models break sharply here.

  • False positives are a major failure mode

    Some models recover many actual error rows but collapse overall by flagging clean derivations at high rates. The benchmark makes that visible fast.

  • Difficulty hurts real verifiers more than weak ones

    GPT-5.4 drops from easy to expert as rule load grows. Weaker models stay flat because they're already near a floor — consistent with the pattern-matching hypothesis.
